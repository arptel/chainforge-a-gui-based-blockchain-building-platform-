Certificate Verification System - Demo Project for ChainForge

=====================================================
OBJECTIVE
=====================================================
Build a working certificate issuance and verification system that acts as a demo application utilizing a simulated ChainForge blockchain. The goal is to prove that ChainForge can power decentralized applications through smart contract interactions.

The demo includes:
1. College Portal (Issuer)
2. Company Portal (Verifier)
3. Student View
4. Backend API (FastAPI)
5. Dummy Blockchain Integration Layer (Mock ChainForge SDK)

=====================================================
ARCHITECTURE
=====================================================
Frontend:
- React + Vite
- Tailwind CSS (Modern, polished UI)

Backend:
- Python FastAPI
- Interfaces directly with a running ChainForge Blockchain Node (deployed from the downloaded ZIP)
- Forwards API endpoints to the blockchain's REST/RPC endpoints

ChainForge Smart Contracts:
- Python-based Smart Contract representing Certificate management
- Written on the ChainForge platform, and incorporated directly into the generated ChainForge Node ZIP
- The demo interacts with the contracts running on this live node

=====================================================
SMART CONTRACT REQUIREMENTS
=====================================================
Contract: CertificateRegistry

Methods:
- issue_certificate(cert_id, student_name, degree, year, issuer_id)
- revoke_certificate(cert_id)
- verify_certificate(cert_id) -> bool
- get_certificate(cert_id)

Rules:
- Only accounts acting as ISSUER can issue certificates.
- Only the original issuer can revoke their certificates.
- Anyone can verify a certificate.
- cert_id must be deterministic and unique.

=====================================================
ROLES
=====================================================
1. ISSUER (College)
   - Authenticates via JWT on backend
   - Can issue and revoke
2. VERIFIER (Company)
   - Verifies certificates by ID
3. STUDENT
   - Views their own digital certificate by ID

=====================================================
BACKEND REQUIREMENTS (FastAPI)
=====================================================
Endpoints:
POST /auth/login (for College/Issuer)
POST /api/issue (Requires JWT)
POST /api/revoke (Requires JWT)
GET  /api/verify/{cert_id}
GET  /api/certificate/{cert_id}

=====================================================
FRONTEND REQUIREMENTS
=====================================================
3 Primary Views:
1. College Dashboard (Protected)
2. Company Verification Page
3. Student Certificate View
