# ChainForge Simulator — How to Run (v2.0)

The ChainForge Simulator is a "Universal Runner": it auto-detects blockchain code generated by the platform and runs it in a real-time, visual peer-to-peer network for testing and demonstration.

---

## 1. Prerequisites
Ensure you have **Python 3.10+** installed.

1. Open a terminal in the `Simulator/backend` folder:
   ```powershell
   cd "Simulator/backend"
   ```
2. Install the required dependencies:
   ```powershell
   python -m pip install -r requirements.txt
   ```

---

## 2. Integrating Platform Downloads (ZIP)
When you build a blockchain on the platform and download the generated `.zip` file:

1. **Locate the ZIP**: Find your downloaded ZIP (e.g., `hotstuff_chain.zip`).
2. **Extract Contents**: Extract everything into this specific folder:
   `Simulator/backend/chain_node/`

   **⚠️ CRITICAL**: The folder must be named exactly `chain_node` and should contain `genesis.json`, `core/`, `modules/`, etc. directly inside it.

   If using PowerShell:
   ```powershell
   mkdir -Force chain_node
   # Use Windows Explorer or 'Expand-Archive' to extract into this folder
   ```
3. **Install Chain Dependencies**:
   Platform-generated code might have its own requirements (e.g., `ecdsa`).
   ```powershell
   python -m pip install -r chain_node/requirements.txt
   ```

---

## 3. Running the Simulator

### Step A: Start the Backend
1. Open a terminal in `Simulator/backend`.
2. Run the server:
   ```powershell
   python main.py
   ```
   *Note: If port 8000 is taken, it will automatically try 8001 or 8002.*

3. **Check the Output**: Look for the log line:
   `Open the dashboard at: http://localhost:8000/static/index.html`

### Step B: Open the Dashboard
1. Copy the URL from Step A:3.
2. Paste it into your browser (Chrome/Edge recommended).
3. **DO NOT** double-click the `index.html` file directly from your folder; it must be served by the backend to communicate correctly.

---

## 4. Using the Simulator

### 🏗️ Network Setup
- **Auto-Detection**: On startup, the simulator detects the `chain_node/` folder. If correctly loaded, the header will say "⛓ ChainForge Simulator".
- **ℹ️ Chain Info**: Click the info icon in the header to see your blockchain's configuration (Consensus type, Network Protocol, Governance mode).
- **+ Add Node**: Click this button to spawn nodes. Choose a role (Validator, Full, Light, etc.) defined in your chain's genesis.
  - **Permissioned/Consortium**: If your chain is consortium-based, adding a node creates a **Join Request**. You must click the original node (Node-0) to approve the request before the new node joins the graph.

### 💸 Transactions & Consensus
- **Node Selection**: Click any node on the whiteboard to open its **Detail Panel**.
- **Send Transaction**: In the detail panel, enter a message/text and click "Send".
- **Consensus Phases**: Watch the bar at the bottom of the whiteboard. You will see real-time phase indicators (e.g., *Prepare -> Pre-commit -> Commit*) and packet animations moving between nodes as they reach consensus.
- **Persistence**: Every block mined is saved to a local SQLite database in `Simulator/backend/data/node_XXX.sqlite`.

### 🔄 Management
- **Reset**: Click "Reset" to clear the network graph and delete all local node databases for a fresh start.
- **Topology**: You can click and drag nodes to rearrange the visual network map.

---

## Troubleshooting
- **No chain_node/ error**: If the screen stays on "No chain_node/ folder detected", ensure you extracted the platform code into the correct path.
- **Port Busy**: Use `$env:PORT=8080; python main.py` to force a specific port.
- **UI glitches**: Use the "Reset" button to clear cached network states.
